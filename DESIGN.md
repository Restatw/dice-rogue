# 骰途 Dice Rogue — 設計文件（GDD）

8-bit 風格、擲骰戰鬥的 Roguelike RPG。單頁、全程跑在 canvas（Phaser 3），支援 PWA 離線安裝。

## 1. 遊戲流程

```
主選單 → 選難度 → 隨機生成 6 名候選角色（選 3 組隊）
       → 大地圖（由下往上的層狀樹，最後收束到 BOSS）
            節點：戰鬥 / 精英 / 事件 / 寶箱 / 休息 / BOSS
       → 戰鬥（擲骰）→ 回地圖 → … → BOSS → 結算
```

對應場景檔：`Boot → MainMenu → Difficulty → PartySelect → Map → Combat → Result`。

## 2. 核心：擲骰戰鬥

我方與敵方每次行動都丟 **5 顆 d6**，總和範圍 **5–30**（平均 17.5，鐘形分布）。
傷害 = `基礎攻擊 × 分級倍率 × 連線倍率 × 職業縮放 × 奇偶剋制`。

### 2-1. 暴擊系統一：依「總和」分級（手感骨幹）

| 總和 | 分級 | 倍率 | 機率（5d6）|
|------|------|------|------|
| 5–7 / 28–30 | 幸運暴擊 | ×3.0 | ~0.5%（傳說一擊）|
| 8–10 / 25–27 | 落空 Miss | ×0 | ~5% |
| 11–15 / 20–24 | 強力擊中 | ×1.5 | ~49% |
| 16–19 | 普通擊中 | ×1.0 | ~41% |

> 已把暴擊從原本只有 5/30 放寬到 5–7 / 28–30；但尾端本就稀薄，所以「總和暴擊」定位為**罕見的大招**。

### 2-2. 暴擊系統二：依「骰型」連線（常見爽快 proc，與總和獨立）

| 骰型 | 額外倍率 |
|------|------|
| 五連（5 同點）| ×4.0 + 特效 |
| 四連（4 同點）| ×2.5 |
| 順子（1-5 或 2-6）| ×2.0 |
| 三連（3 同點）| ×1.5 |

≥3 同點在 5d6 約 **21%**，因此這是玩家每場都會遇到的興奮點。兩套暴擊「類型不同」：一個看總和、一個看骰型，可疊加。

### 2-3. 元素屬性與剋制（風火水土）

底層規則是骰子總和的「**奇偶 × 大小**」兩個隱藏軸（玩家不直接看到奇偶），交叉成四象限元素：

| | 小（S ≤ 17） | 大（S ≥ 18） |
|---|---|---|
| **奇數** | 🟢 風 wind | 🟤 土 earth |
| **偶數** | 🔵 水 water | 🔴 火 fire |

- 角色 / 怪物生成時擲骰，**總和＝能力編碼**，由編碼推出元素屬性（顯示為風火水土）。
- 每次攻擊由「該次骰的總和」得出攻擊元素，與防守方元素依**循環剋制**比對：

**循環：水 → 火 → 風 → 土 → 水**（箭頭＝克制）

| 關係 | 倍率 |
|---|---|
| 攻方克防方 | ×1.5（增傷）|
| 攻方被防方克 | ×0.5（減傷）|
| 其餘（同屬/不相鄰）| ×1.0（普通）|

（資料在 `src/data/elements.js` 的 `beats`，倍率在 `balance.js` 的 `element`。遊戲內主選單「屬性表」可查相剋表。）

> 註：因奇偶與大小並非完全獨立，靠近中央的總和(17/18)落在風/火，故風/火(~28%)略多於土/水(~22%)，視為元素稀有度差異。

### 2-4. 職業縮放（四職業看不同骰象）

| 職業 | 看什麼 | 效果 |
|------|--------|------|
| 戰士 | 總和大小 | 總和越高傷害越高（×0.8→1.5），高血前排 |
| 法師 | 偶數骰數量 | 每顆偶數 +0.12，偶數爆發 |
| 盜賊 | 奇數骰數量 + 連線 | 奇數加成，連線時再 ×1.2，連線專家 |
| 祭司 | 總和奇偶 | 偶=輸出；奇=改為治療最虛弱隊友 |

> 全部寫成 `scale(ctx)` 純函式（`src/data/classes.js`），數值可調；事件/道具可在 runtime 覆寫 `BALANCE` 副本來改規則。

## 3. 大地圖

- 由下往上的層狀樹（預設 12 層），每層 2–4 節點，邊連到上一層鄰近節點並逐步收束到單一 BOSS。
- 節點型別權重隨深度調整（後段精英變多、中段休息點增加）。
- 生成可重現：整個 run 共用一個 seed（`src/core/rng.js` mulberry32），可分享種子。

## 4. 技術架構

```
src/
  main.js              Phaser game 設定 + 場景註冊 + PWA 註冊
  core/                純邏輯（不依賴 Phaser，可單測）
    rng.js             seeded RNG
    dice.js            擲骰 / 分級 / 連線
    combat.js          一次攻擊的完整結算
    party.js           候選角色生成
    mapgen.js          地圖樹生成
    runState.js        一次冒險的全域狀態
  data/                資料驅動的數值
    balance.js         所有可調數字（分級/連線/奇偶/難度/地圖）
    classes.js         四職業 + scale()
    enemies.js         怪物模板 + 生成
  scenes/              Phaser 場景（畫面）
  ui/widgets.js        共用 8-bit UI 元件
scripts/sim.js         無瀏覽器的平衡驗證（npm run sim）
```

**設計原則**：邏輯與畫面分離。`core/` + `data/` 完全不 import Phaser，所以戰鬥平衡可用 `npm run sim` 在 Node 直接驗證，也方便日後寫單元測試。

## 5. 開發里程碑（建議）

- **M1（已完成骨架）**：流程串接、可重現 RNG、戰鬥引擎、地圖生成、PWA。
- **M2**：戰鬥演出（骰子滾動動畫、傷害數字、震動）、敵方多單位、目標選擇。
- **M3**：道具/技能/詛咒系統（覆寫 BALANCE）、商店、事件分支。
- **M4**：8-bit 美術（點陣字、角色/怪物 spritesheet、tilemap 背景）、音效/BGM。
- **M5**：存檔（localStorage/IndexedDB）、種子分享、成就。

## 6. 待你拍板的平衡點

- 幸運暴擊倍率 3.0、連線倍率、奇偶 0.5/1.5 都是草案 → 跑 `npm run sim` 看分布後微調。
- 祭司「奇數=治療、輸出歸 0」較極端，可能改成「奇數=半輸出+小治療」。
- 怪物是否也吃連線/職業類規則，或保持單純倍率。
